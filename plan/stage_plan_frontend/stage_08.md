# Stage 08: 认证与权限体验（前端侧）

## 目标
补齐前端的认证体验（登录/注册/退出、受保护路由、权限提示），并把“鉴权失败”从 silent failure 变成清晰的用户流程。

## Todo List
- [x] 增加 Auth 页面：`/login`、`/register`（表单校验、错误提示、loading 状态）。
- [x] 增加 token 管理：安全存储策略评估、退出登录、401 自动处理（清 token + 引导登录）。
- [x] 路由保护：对 `/projects`、`/editor/*` 等受保护页面做 guard（未登录重定向）。
- [x] Navbar 用户区：显示当前用户、下拉菜单（Profile/Logout），替换占位头像。
- [x] 权限与空状态：无权限/资源不存在/未公开项目的统一提示页。

## 验收标准
- 未登录访问受保护页面会被正确引导到登录页，登录后可返回原目标页。
- 401/403 的错误不再只是 console 输出，而是明确的 UI 提示与可操作入口。
