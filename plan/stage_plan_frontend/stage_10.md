# Stage 10: 性能、可访问性与设计收尾

## 目标
对全站进行“体验收尾”：性能、可访问性（a11y）、交互细节与视觉一致性，让产品达到可演示与可长期维护的水准。

## Todo List
- [x] 性能：减少不必要的 client components、按需动态加载、ReactFlow/Timeline 性能 profiling。
- [x] 资源：统一使用 Next/Image（如适用）、为视频/缩略图建立资源占位与加载策略。
- [x] 可访问性：键盘可操作、focus 管理、语义化标签、对比度与 aria 属性。
- [x] 设计收尾：页面间动效一致、空状态/错误态细节、图标与排版统一。
- [x] 文档：补齐前端开发指南（目录结构、组件约定、API/hooks 使用方式）。

## 验收标准
- Lighthouse（或同等检查）在核心页面达到可接受指标（性能/可访问性显著提升）。
- 关键交互可用键盘完成，视觉与动效整体一致。
